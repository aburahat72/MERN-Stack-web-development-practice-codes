# Why React
- We can use the states which means that once we update the state variable, it changes across the pages
- we can split our app into multiple components and reuse those components
- React uses a virtual DOM to effectively update the UI which is better than updating content using the DOM Manipulation.
- Debugging and maintainance is easy.

# States
- state is a built-in object used to store data or information about a component that can change over time. It acts like a component's personal memory. When a user interacts with your application (e.g., clicking a button, typing in a form), the state changes. The most crucial aspect of state is that whenever it updates, React automatically re-renders the component to update the user interface (UI) on the screen
